# RunPick UI/UX 개선 실행 계획 (v1)

## 0. 목적

이 문서는 현재 RunPick의 사용자경험에서 이탈/혼란을 유발하는 지점을 정리하고, 코드베이스 기준으로 바로 실행 가능한 개선 계획을 제시합니다.

핵심 목표:

1. 주요 전환 경로(홈 -> 카탈로그/퀴즈 -> 상세 -> 구매 유도)의 마찰 감소
2. 탐색 맥락(스크롤 위치, 필터 상태, 진행 상태) 유지
3. 접근성/모션 과부하 이슈 완화
4. 개선 효과를 지표로 검증 가능한 상태 구축

## 2. 사용자 여정 기준 개선 포인트

### 2.1 홈 -> 탐색 진입

- 문제: 시각 효과 대비 행동 유도가 분산됨(퀴즈 vs 카탈로그 우선순위가 상황별로 불명확).
- 조치: 홈에서 사용 목적(빠른 추천/직접 탐색) 선택을 더 명확히 분기.

### 2.2 카탈로그 탐색

- 문제: 스크롤 위치/필터 상태 유지가 약하고, 빠른 후보 축약 도구(검색/정렬/비교)가 없음.
- 조치: URL 기반 상태, 검색/정렬, 임시 비교 보드 도입.

### 2.3 상세 -> 전환

- 문제: 가장 강한 CTA가 dead-end.
- 조치: 데이터 모델에 공식 구매 링크 추가 후 CTA 연결.

### 2.4 퀴즈

- 문제: 자동진행 중심 상호작용이 오입력 복구 비용을 높임.
- 조치: 명시적 `다음` 버튼 모드(기본) + 자동진행은 옵션화.

### 2.5 공통 경험

- 문제: 모션/커서/핀 스크롤 효과가 사용자 선호를 반영하지 못함.
- 조치: `prefers-reduced-motion` 및 설정 기반 효과 강도 제어.

## 3. 실행 계획 (Phase별)

## Phase 0. 측정 기반 준비 (0.5주)

목표: 개선 전후 비교가 가능한 계측 최소 단위 확보.

작업:

3. 페이지별 핵심 액션에 이벤트 삽입

이벤트 초안:

- `home_cta_click` (`quiz_start`, `catalog_start`)
- `catalog_filter_change`
- `shoe_card_click`
- `shoe_detail_store_click`
- `quiz_start`, `quiz_answer`, `quiz_complete`
- `quiz_result_primary_click`, `quiz_result_alt_click`

완료 기준:

- 로컬에서 이벤트 payload 로그/검증 가능
- 이벤트 명세 문서화 완료

## Phase 1. P0 즉시 개선 (1~1.5주)

### 1) 상세 페이지 dead CTA 해소

수정:

- `src/types/shoe.ts`: `officialUrl?: string` 필드 추가
- `src/data/brands/asics.json`: 각 모델에 공식 URL 추가
- `src/app/shoe/[shoeId]/ShoeDetailClient.tsx`: `공식 스토어에서 보기` 버튼에 `href={shoe.officialUrl}` 연결
- URL이 없는 경우 버튼을 `입고/링크 준비중` 상태로 비활성 처리

완료 기준:

- 모든 상세 페이지 CTA 클릭 시 유효한 외부 링크 또는 명확한 대체 상태 제공
- dead click 0건

### 2) 목록 맥락 보존(스크롤 복원)

수정:

- `src/components/layout/SmoothScroll.tsx`
- 현재 pathname 변경 시 강제 `scrollTo(0)` 로직 제거
- 브라우저 뒤로가기는 기존 스크롤 복원, 명시적 새 진입만 상단 이동
- 필요 시 `sessionStorage` 기반 pathname+search 단위 복원

완료 기준:

- `브랜드 목록 -> 상세 -> 뒤로가기`에서 이전 스크롤 위치 복귀
- 재탐색 체감 시간 감소(정성 테스트)

### 3) 퀴즈 오입력 완화

수정:

- `src/components/quiz/QuizQuestion.tsx`
- `src/app/quiz/page.tsx`

옵션(권장):

- 기본: 선택 후 자동진행 제거, `다음` 버튼으로 진행
- 보조: `자동으로 다음 질문` 토글(선택 사항)

완료 기준:

- 오입력 후 즉시 수정 가능한 UI 제공
- 퀴즈 완주율 +5%p 목표

## Phase 2. P1 핵심 사용성 강화 (2주)

### 4) URL 기반 필터 상태 동기화

수정:

- `src/app/brand/[brandId]/[category]/CategoryPageClient.tsx`

내용:

- 잘못된 값은 `all`로 정규화
- 공유/새로고침/뒤로가기 시 동일 상태 복원

완료 기준:

- 카테고리 페이지 URL만으로 동일 화면 재현 가능

### 5) 카탈로그 탐색 도구 추가 (검색/정렬)

수정 후보:

- 신규 `src/components/catalog/CatalogToolbar.tsx`
- `BrandPageClient.tsx`, `CategoryPageClient.tsx`

기능:

- 검색: 모델명(한/영), 카테고리/서브카테고리 키워드
- 정렬: 추천순(기본), 가벼운순, 쿠셔닝 높은순, 가격 낮은순
- 결과 개수/필터 리셋 표시

완료 기준:

- 12개 이상 데이터에서도 원하는 모델 도달 클릭 수 감소

### 6) 접근성 핵심 보완

수정:

- `src/components/layout/Header.tsx`: `aria-expanded`, `aria-controls`, 메뉴 포커스 트랩/ESC 닫기
- `src/components/common/Button.tsx`: 링크형 버튼의 `focus-visible` 스타일 보장
- `src/app/layout.tsx`: `skip to content` 링크 추가

완료 기준:

- 키보드만으로 `홈 -> 카탈로그 -> 상세 -> 뒤로` 흐름 수행 가능
- 치명적 접근성 이슈 0건

### 7) 모션 강도 제어 (Reduced Motion)

수정:

- `src/app/globals.css`: `@media (prefers-reduced-motion: reduce)` 정책 추가
- `src/components/layout/CustomCursor.tsx`, `FeaturedShoes.tsx`, `HeroSection.tsx` 등 주요 효과 컴포넌트에서 모션 축소 분기

완료 기준:

- OS의 reduced motion 설정 시 과도한 애니메이션/커서 효과 비활성화

## 4. 검증 지표 (KPI)

## 퍼널

- 홈 -> 퀴즈 시작 클릭률: +15%
- 퀴즈 완주율: +5%p
- 결과 -> 상세 진입률: +10%
- 상세 -> 스토어 클릭률: dead click 제거 후 정상 분모 기준 측정

## 행동 품질

- 목록 페이지 평균 재탐색 시간(상세에서 뒤로 복귀 후 다음 클릭까지): 감소
- 필터/정렬 사용률: 증가
- 카테고리 페이지 이탈률: 감소

## 기술 품질

- 모바일 INP 개선
- 상위 랜딩 LCP 안정화
- 접근성 critical 이슈 0건

## 5. 릴리즈 전략

1. Phase 1(P0)은 기능 플래그 없이 바로 반영 가능한 범위부터 선반영
2. Phase 2는 검색/정렬과 접근성 보완을 묶어 배포
3. Phase 3는 비교 기능을 실험 플래그로 점진 배포

권장 배포 단위:

- 배포 A: CTA 연결 + 스크롤 복원 + 퀴즈 진행 방식 개선
- 배포 B: URL 필터 + 검색/정렬 + 접근성
- 배포 C: 헤더/탭 통일 + 비교 기능
